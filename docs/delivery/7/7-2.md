## Requirements

1. Create helper functions in `internal/modules/devtools/core.go`:
   - `installCoreTools(cfg *config.Config) error` - Installs Git, build-essential, curl, wget, ca-certificates
   - `coreToolsInstalled() (bool, error)` - Checks if all core tools are installed
   - Individual check functions: `gitInstalled()`, `buildEssentialInstalled()`, etc.

2. Install packages via apt:
   - `git` - Version control
   - `build-essential` - GCC, make, and other build tools
   - `curl` - HTTP client
   - `wget` - File downloader
   - `ca-certificates` - SSL certificates

3. Idempotency:
   - Check if packages are already installed before installing
   - Use `dpkg -l | grep` or `which` to check installation
   - Return early if all tools are already installed

4. Error handling:
   - Handle apt update errors
   - Handle package installation errors
   - Return descriptive errors

5. Logging:
   - Use `log.Info()` for progress messages
   - Use `log.Success()` when installation completes
   - Use `log.Skip()` when tools are already installed

6. Dry-run support:
   - Check dry-run mode from logging package
   - Log what would be done but don't execute commands

## Implementation Plan

4. Add proper error handling and logging
