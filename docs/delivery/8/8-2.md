## Requirements

1. Load config file using path from `--config` Cobra flag (default: "config.yaml")
2. Use `config.Load()` function from `internal/config` package
3. Handle file not found errors gracefully
4. Handle validation errors with clear messages
5. Return config or error to caller
6. Log errors using `log.Error()`
7. If config file doesn't exist, use default config (via `config.DefaultConfig()`)

## Implementation Plan

1. Create `loadConfig(path string) (*config.Config, error)` function in `main.go`
2. Check if file exists using `exec.FileExists()` or `os.Stat()`
3. If file doesn't exist:
   - Log info message that default config will be used
   - Return `config.DefaultConfig()` (but still need to validate required fields)
   - OR: Return error if config file is required
4. If file exists:
   - Call `config.Load(path)`
   - Handle errors:
     - File read errors: return with clear message
     - YAML parse errors: return with clear message
     - Validation errors: return with clear message
5. Log success message when config loads
6. Call `loadConfig()` from `runCommand()` when needed (not in list mode)

## Implementation Notes

### Implementation Details

1. **Config loading function**:
   - Created `loadConfig(path string) (*config.Config, error)` function in `main.go`
   - Uses `exec.FileExists()` to check if config file exists before attempting to load
   - If file doesn't exist: Returns `config.DefaultConfig()` with warning message
   - If file exists: Calls `config.Load(path)` and handles errors appropriately

2. **Error handling**:
   - File not found: Returns default config with warning (as per requirement 7)
   - Invalid YAML: Returns error with clear message showing parse error
   - Validation errors: Returns error with clear message showing which field is missing
   - All errors are wrapped with context (file path)

3. **Integration**:
   - `loadConfig()` is called from `runCommand()` after flag validation
   - Config is loaded before profile/module selection (will be used in subsequent tasks)
   - Config loading is skipped when `--list` flag is set (not needed for listing)

4. **Logging**:
   - Warning logged when config file not found and defaults are used
   - Info message logged when config loads successfully
   - Error messages logged via `log.Error()` for all error cases
