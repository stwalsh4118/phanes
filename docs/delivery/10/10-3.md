## Requirements

1. Create `internal/runner/summary.go` file
2. Implement `PrintSummary(results []ModuleResult, dryRun bool)` function
3. Format as table using box drawing characters (as shown in PRD)
4. Color-coded status indicators:
   - Green for installed (✓ Installed)
   - Yellow for skipped (⊘ Skipped)
   - Red for failed/error (✗ Failed / ✗ Error)
5. Show error messages for failed modules in Details column
6. Display totals: "X installed, Y skipped, Z failed"
7. Handle dry-run mode indicator (show "(dry-run)" if applicable)

## Implementation Plan

1. Create `internal/runner/summary.go` file
2. Define ANSI color constants (green, yellow, red, reset)
3. Implement `PrintSummary` function:
   - Calculate column widths (Module name, Status, Details)
   - Print table header with box drawing characters
   - For each result, print row with appropriate color and status symbol
   - Print error messages in Details column for failed/error statuses
   - Print table footer
   - Print summary totals line
   - Print dry-run indicator if applicable
4. Use fmt.Printf for formatted output with ANSI colors

## Verification
